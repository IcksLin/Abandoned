```
摄像头图像(160×120 RGB)
    ↓

    ↓

    ↓

    ↓

    ↓

    ↓

    ↓
角度计算 → dangle_Lline[], dangle_Rline[]
    ↓

    ↓
角点识别 → Lline_pt, Rline_pt (LOST/S/T/L/Y类型)
    ↓
状态机决策 → 选择 Mline (L2Mline 或 R2Mline)
    ↓
中线跟踪 → Mline[x,y] (最终中线点集)
    ↓
预瞄点计算 → aim_point.angle (转向角度)
    ↓
PID控制输出
```

## 2. 工作流程详解

### 2.1 tracking() - 主循环函数

**位置**：`my_task_function.cpp:73`

```cpp
void tracking()
{
    //图像获取与处理
    image_proc();
    gray_img_with_centerline_transmitter(img_gray, UVC_WIDTH, IMG_H, Mline, Mline_num, true, true);
}
```

**功能说明**：
1. **调用频率**：每帧图像到达时执行一次（约60fps）
2. **主要任务**：
   - 调用 `image_proc()` 完成图像处理和决策
   - 调用 `gray_img_with_centerline_transmitter()` 将处理结果（带中线的灰度图）通过TCP传输到上位机

**输出结果**：
- `Mline[Mline_num][2]`：最终选择的中线点集
- `aim_point.angle`：转向角度（用于车辆控制）
- `aim_point.flag`：是否有效巡线（0=无效，1=有效）

### 2.2 image_proc() - 图像处理总控

**位置**：`decision.cpp:25`

```cpp
void image_proc(){   
    /*----------图像处理----------*/
    // 等待采样数据
    uvc.wait_image_refresh();
    img_gray = uvc.get_gray_image_ptr();

    start_thre = get_otsu_thres(img_gray,0,IMG_W,TRACK_HEIGHT_MAX,IMG_H);      // 二值化
    //状态机决策
    sta_decision();
}
```

**执行步骤**：
1. **等待图像**：阻塞等待摄像头新帧
2. **获取灰度图**：从UVC设备获取160×120灰度图像指针
3. **动态二值化**：使用大津法（OTSU）计算自适应阈值 `start_thre`
   - 计算范围：从 `TRACK_HEIGHT_MAX`(40行) 到图像底部
   - 目的：根据当前光照环境自动调整黑白分割阈值
4. **决策执行**：调用 `sta_decision()` 进行状态机决策

## 3. 图像处理模块（imgproc）

### 3.1 核心处理流程

#### 🔹 line_process() - 边线处理总控

**位置**：`imgproc.cpp:112`

**函数原型**：
```cpp
void line_process(uint8_t height_start, uint8_t height_min)
```

**参数说明**：
- `height_start`：巡线起始高度（通常为 `IMG_H=120`）
- `height_min`：巡线最低高度（通常为 `IMG_H/2=60` 或 `TRACK_HEIGHT_MAX=40`）

**处理流程**：

```cpp
// 1. 左右巡线
search_Lline(height_start, height_min);  // 寻找左边线
search_Rline(height_start, height_min);  // 寻找右边线

// 2. 判断是否丢线（线长<LINE_LOST_LENGTH=16）
if (Lline_num <= LINE_LOST_LENGTH)
    sampled_Lline_num = 0;  // 标记为丢线
else {
    // 3. 透视变换：将图像坐标转换为俯视图坐标
    perspective_transform_points(Lline, Lline_num, per_Lline);
    
    // 4. 三角滤波：平滑边线噪声
    blur_points(per_Lline, Lline_num, blurred_Lline);
    
    // 5. 等距采样：将不均匀点集转为固定距离采样点
    resample_points(blurred_Lline, Lline_num, sampled_Lline, &sampled_Lline_num);
    
    // 6. 角度计算：计算每个点的局部曲率
    local_angle_points(sampled_Lline, sampled_Lline_num, dangle_Lline);
    
    // 7. 非极大抑制：找出最大角度点（角点）
    nms_angle(dangle_Lline, sampled_Lline_num, &nms_Lline, &nms_Lline_idx);
}

// 右边线同样处理
```

---

### 3.2 关键函数详解

#### 🔹 search_Lline() / search_Rline() - 迷宫巡线

**算法**：左手/右手法则自适应巡线

**核心思想**：
1. **起始点查找**：
   - 从图像左侧（右侧）底部开始
   - 沿x轴向内搜索白色区域（赛道内部）
   - 找到黑白边界作为起始点

2. **自适应阈值**：
   - 每个点计算局部阈值（`all_block_size×all_block_size` 窗口平均）
   - 减去偏差值 `adapt_clip=7` 提高鲁棒性
   - 局部阈值避免全局光照不均影响

3. **巡线规则**：
   ```
   左手法则：优先往左前方走，其次正前方，最后左转弯
   右手法则：优先往右前方走，其次正前方，最后右转弯
   ```

4. **终止条件**：
   - 达到最大点数 `POINTS_MAX_LEN=100`
   - 出界（超出图像范围）
   - 连续转向4次（可能进入死循环）

**输出**：
- `Lline[Lline_num][2]` / `Rline[Rline_num][2]`：边线点集（图像坐标）
- `Lline_num` / `Rline_num`：边线点数

---

#### 🔹 perspective_transform_points() - 透视变换

**目的**：将摄像头拍摄的斜视图转换为俯视图

**原理**：
- 使用预先标定的透视矩阵 `M` (3×3)
- 通过查找表 `map_x[y][x]`, `map_y[y][x]` 快速映射
- 避免每次计算矩阵乘法，提高效率

**效果**：
- 直线在俯视图中仍为直线
- 距离关系更符合真实世界
- 便于后续几何计算

---

#### 🔹 blur_points() - 三角滤波

**目的**：平滑边线，去除巡线噪声

**算法**：
```cpp
for (int i = 0; i < num; i++) {
    float sum_x = 0, sum_y = 0, weight_sum = 0;
    int half = all_block_size / 2;
    
    for (int j = max(0, i-half); j <= min(num-1, i+half); j++) {
        float weight = 1.0 - abs(i-j) / (float)(half+1);  // 三角权重
        sum_x += pts_in[j][0] * weight;
        sum_y += pts_in[j][1] * weight;
        weight_sum += weight;
    }
    
    pts_out[i][0] = sum_x / weight_sum;
    pts_out[i][1] = sum_y / weight_sum;
}
```

**特点**：
- 中心点权重最大，两侧递减
- 窗口大小为 `all_block_size=7`
- 既平滑又保留边线形状

---

#### 🔹 resample_points() - 等距采样

**目的**：将不均匀的边线点转换为固定距离采样点

**参数**：
- `sampled_dist = 0.02f` 米（采样间距）
- `M2PIX = 100`：米到像素转换系数
- 实际采样距离 = 2 像素

**算法思想**：
```
沿着折线段行走，每走够固定距离就采样一个点
```

**优点**：
- 统一点密度，便于后续角度计算
- 减少点数，提高处理速度
- 避免点集密度不均影响算法

---

#### 🔹 local_angle_points() - 局部角度计算

**目的**：计算每个点的局部曲率变化率

**算法**：
```cpp
// 对第 i 个点
向量1 = pts[i] - pts[i-angle_idx]  // 前向向量
向量2 = pts[i+angle_idx] - pts[i]  // 后向向量

angle = arccos(向量1·向量2 / (|向量1|×|向量2|))  // 夹角
```

**参数**：
- `angle_idx = 10`：采样索引距离
- 实际距离 = 10 × 2像素 = 20像素 ≈ 0.2米

**输出含义**：
- `angle ≈ 0°`：直线段（向量同向）
- `angle ≈ 90°`：L型角点（向量垂直）
- `angle ≈ 180°`：Y型角点（向量反向）

---

#### 🔹 nms_angle() - 非极大抑制

**目的**：找出角度变化率最大的点（角点）

**算法**：遍历所有点，记录最大角度值及其索引

**输出**：
- `nms_Lline`：最大角度值（度数）
- `nms_Lline_idx`：角点在采样点集中的索引

**角点判断标准**：
```cpp
nms_val ≤ 15°      → 直道 (S_PT)
15° < nms_val ≤ 75° → 弯道 (T_PT)
75° < nms_val ≤ 93° → L角点 (L_PT)
nms_val > 93°       → Y角点 (Y_PT)
```

---

### 3.3 中线跟踪函数

#### 🔹 track_leftline() / track_rightline()

**目的**：从边线推算中线

**算法**：
```cpp
void track_leftline(float dist = ROAD_W*M2PIX/2) {
    for (int i = 0; i < sampled_Lline_num; i++) {
        // 计算边线在该点的法向量（垂直方向）
        向量 = sampled_Lline[i+1] - sampled_Lline[i-1]
        法向量 = (-向量.y, 向量.x) / |向量|
        
        // 沿法向量向右偏移半个赛道宽度
        L2Mline[i] = sampled_Lline[i] + 法向量 × dist
    }
}
```

**参数**：
- `dist`：偏移距离，默认为半个赛道宽度（22.5像素）
- `ROAD_W = 0.45f` 米（赛道宽度）

**作用**：
- 左边线向右偏移 → 得到中线
- 右边线向左偏移 → 得到中线

## 4. 决策模块（decision）

### 4.1 角点识别系统

#### 🔹 PT_Judge_TypeDef - 角点类型枚举

```cpp
typedef enum {
    LOST_PT = 0x01,    // 丢线     000001
    S_PT    = 0x02,    // 直道     000010
    T_PT    = 0x04,    // 弯道     000100
    L_PT    = 0x08,    // L角点    001000
    Y_PT    = 0x10,    // Y角点    010000
    INVAL_PT= 0x20,    // 无效角点 100000
} PT_Judge_TypeDef;
```

**设计思想**：使用位标志，便于组合判断

**组合掩码**：
```cpp
SUPPABLE_PT = L_PT | Y_PT           // 可补线的边线
NORMAL_PT   = S_PT | T_PT           // 普通边线
TRABLE_PT   = S_PT | T_PT | L_PT | Y_PT  // 可巡线的边线
INTRABLE_PT = LOST_PT | INVAL_PT    // 不可巡线的边线
```

**使用示例**：
```cpp
if (Lline_pt & NORMAL_PT)  // 判断左边线是否为普通边线
if (Rline_pt & SUPPABLE_PT) // 判断右边线是否可补线
```

#### 🔹 pt_identify() - 角点识别函数

**位置**：`decision.cpp:39`

**输入**：
- `nms_val`：角度变化率最大值
- `linelen`：原始线长

**判断逻辑**：
```cpp
// 1. 丢线判断
if (linelen < LINE_LOST_LENGTH=16)
    pt = LOST_PT;

// 2. 直道判断
else if (nms_val ≤ STRAIGHT_UP=15°) {
    if (linelen < LINE_ST_MIN_LEN=25)
        pt = INVAL_PT;  // 太短，无效
    else
        pt = S_PT;      // 直道
}

// 3. 弯道判断
else if (nms_val ≤ LPT_LOW=75°)
    pt = T_PT;

// 4. L角点判断
else if (nms_val ≤ LPT_UP=93°)
    pt = L_PT;

// 5. Y角点判断
else if (nms_val > LPT_UP=93°)
    pt = Y_PT;
```

**稳定性处理**：
```cpp
// 连续判断 __JUDGE_CNT=2 次相同结果才切换
if (pt_cnt >= __JUDGE_CNT) {
    out = pt;       // 输出新结果
    pt_cnt = 1;     // 重置计数
}
else {
    if (pt != per_pt) pt_cnt = 1;  // 结果变化，重新计数
    else pt_cnt++;                 // 结果相同，计数+1
}
```

**防抖动作用**：避免因图像噪声导致角点类型频繁切换

#### 🔹 sta_decision() - 状态机决策引擎

**位置**：`decision.cpp:215`

**核心代码**：
```cpp
void sta_decision(void){
    // 1. 默认不巡线（保持上次角度）
    aim_point.flag = 0;
    
    // 2. 执行当前状态处理函数
    cur_sta->func();
    
    // 3. 根据决策结果确定最终巡线方式
    if (aim_point.flag) {
        if (Mline == L2Mline && (Lline_pt & TRABLE_PT)) {
            // 巡左边线
            Mline_num = sampled_Lline_num;
            track_leftline();
        }
        else if (Mline == R2Mline && (Rline_pt & TRABLE_PT)) {
            // 巡右边线
            Mline_num = sampled_Rline_num;
            track_rightline();
        }
        else {
            // 无法巡线
            aim_point.flag = 0;
        }
    }
}
```

**工作流程**：
```
执行状态函数 → 设置 Mline 指针 → 验证边线有效性 → 生成中线 → 计算预瞄点
```

### 4.3 状态处理函数详解

#### 🔹 normal_proc() - 正常巡线

**位置**：`decision.cpp:233`

```cpp
// 1. 边线处理（全图搜索）
```

**决策规则**：

|--------------|--------------|---------------|
| LOST/INVAL   | S_PT/T_PT    | 巡右边线       |

| Y_PT         | 任意         | 切换到左圆环   |

**线长切换逻辑**：
```cpp
if (Mline == L2Mline) {  // 当前巡左边线
    if (Rline_num > Lline_num + __N_DFT_LEN_THRE=30)
        Track_Rline();  // 右边线更长，切换

else {  // 当前巡右边线
    if (Lline_num > Rline_num + __N_DFT_LEN_THRE=30)
        Track_Lline();  // 左边线更长，切换
}
```

**作用**：
- 避免频繁切换（加入30个点的阈值）
- 优先选择长边线（信息更可靠）

#### 🔹 cross_proc() - 十字处理

**位置**：`decision.cpp:295`

**特点**：
- 限制搜索高度（`IMG_H * 3/4`），避免误检测远处线
- 优先补左边线（人为设定）
- 驶出条件：左右均为普通边线

**执行流程**：
```cpp
// 1. 边线处理（限制高度）
line_process(IMG_H, IMG_H*3/4);

// 2. 角点识别
lines_ptIdentify();

// 3. 判断是否驶出
if ((Lline_pt & NORMAL_PT) && (Rline_pt & NORMAL_PT))
    Cur_Sta_SetNormal();  // 恢复正常巡线

// 4. 否则补线巡线
else if (Lline_pt & SUPPABLE_PT) {
    Supplement_Lline();  // 补左边线
    Track_Lline();
}
else if (Rline_pt & SUPPABLE_PT) {
    Supplement_Rline();  // 补右边线
    Track_Rline();
}
```

**补线原理**：
- 计算角点前向量的平均角度
- 根据角度象限判断补线方向（向上/向左/向右）
- 延伸固定长度形成新边线

#### 🔹 Rcircle_proc() - 右圆环处理

**位置**：`decision.cpp:486`

**圆环内部状态机**：
```cpp
typedef enum{
    init_circle,    // 初始化
    begin_circle,   // 准备驶入
    into_circle,    // 驶入中
    in_circle,      // 环内巡线
    out_circle,     // 驶出中
    outof_circle    // 已驶出
} Circle_Flag_TypeDef;
```

**状态转换图**：
```
init_circle (初始化参数)
    ↓
begin_circle (发现Y角点，等待左边线为直道)
    ↓ 条件：Rline_pt==T_PT
into_circle (左边丢失计数，等待进入)
    ↓ 条件：__begin_lose_l >= __RC_BEGIN_LOSE_THRE=1
in_circle (环内巡左边线)
    ↓ 条件：Lline_pt==T_PT && sampled_Lline_num<某值
out_circle (驶出，给固定角度)
    ↓ 条件：发现右边Y角点
outof_circle → 切换到 normal_sta
```

**关键技巧**：
1. **防误判**：如果在 `begin_circle` 发现长直道，立即返回正常状态
2. **丢线处理**：驶入阶段左边丢失时给固定角度 `__RC_OUT_FIXED_ANGLE=20°`
3. **减速标志**：设置 `circle_flag=1`，通知控制系统降速

**左圆环类似，方向相反**

## 6. 状态机详细分析

### 6.1 状态转换关系图

```
                    ┌──────────────┐
                    │  normal_sta  │
                    │   正常巡线   │
                    └──────┬───────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  cross_sta   │  │ Lcircle_sta  │  │ Rcircle_sta  │
│  十字处理    │  │  左圆环      │  │  右圆环      │
└──────┬───────┘  └──────┬───────┘  └──────┬───────┘
        │                  │                  │
        └──────────────────┼──────────────────┘
                           ▼
                    ┌──────────────┐
                    │  normal_sta  │
                    │   恢复正常   │
                    └──────────────┘
```

### 6.2 状态切换条件

#### 从 normal_sta 切换

| 目标状态    | 触发条件                           |
|-----------|-----------------------------------|
| cross_sta | `Lline_pt==L_PT && Rline_pt==L_PT` |
| Rcircle_sta | `Rline_pt==Y_PT`                 |
| Lcircle_sta | `Lline_pt==Y_PT`                 |

#### 从 cross_sta 切换

| 目标状态    | 触发条件                           |
|-----------|-----------------------------------|
| normal_sta | `(Lline_pt & NORMAL_PT) && (Rline_pt & NORMAL_PT)` |

#### 从圆环状态切换

| 阶段         | 切换到 normal_sta 的条件          |
|-------------|----------------------------------|
| begin_circle | 发现长直道（防误判）              |
| outof_circle | 完成圆环流程                     |

---

### 6.3 圆环处理详细流程（右圆环示例）

```
【begin_circle】准备驶入
    - 左边线：S/T 正常巡线
    - 右边线：Y角点 → 补线巡线
    - 等待：右边线变为T弯道
    ↓
【into_circle】驶入中
    - 巡右边线（优先）
    - 左边线丢失计数 __begin_lose_l
    - 计数达到阈值 → 进入下一阶段
    ↓
【in_circle】环内巡线
    - 搜索范围：全图
    - 巡左边线（环内边线）
    - 等待：左边线出现特征（线长变短且为T弯道）
    ↓
【out_circle】驶出中
    - 限制搜索高度（防止远处干扰）
    - 左边线丢失 → 给固定角度
    - 等待：发现右边Y角点
    ↓
【outof_circle】完成
    - 切换到 normal_sta
    - circle_flag = 0（恢复正常速度）
```

## 7. 常见问题与调试

### 7.1 常见问题

#### ❓ 问题1：巡线不稳定，频繁左右摇摆

**可能原因**：
- 二值化阈值不合适
- 预瞄点距离过近
- PID参数调试不当

**解决方案**：
```cpp
// 1. 调整自适应参数
all_block_size = 9;  // 增大窗口，提高稳定性
adapt_clip = 10;     // 增大偏差，降低敏感度

// 2. 增大预瞄点索引
aim_point.idx = 40;  // 从30增加到40

// 3. 检查PID参数（见my_pid模块）
```

---

#### ❓ 问题2：十字路口冲出赛道

**可能原因**：
- L角点判断阈值过高，未识别到
- 补线方向错误

**解决方案**：
```cpp
// 1. 降低L角点阈值
#define LPT_LOW  70  // 从75降低到70

// 2. 检查补线效果
// 在 supplement_line() 中打印 avg_angle
std::cout << "avg_angle: " << avg_angle << std::endl;
```

---

#### ❓ 问题3：圆环识别失败

**可能原因**：
- Y角点阈值过高
- 圆环状态机参数不匹配当前场地

**解决方案**：
```cpp
// 1. 降低Y角点阈值
#define LPT_UP  90  // 从93降低到90

// 2. 调整圆环状态转换条件
#define __RC_BEGIN_LOSE_THRE 2  // 增加丢失计数阈值

// 3. 调整驶出固定角度
#define __RC_OUT_FIXED_ANGLE 15  // 根据实际情况调整
```

## 8. 总结

### 8.1 系统特点

✅ **优点**：
- 自适应二值化，适应光照变化
- 透视变换，提高几何计算准确性
- 状态机架构，处理复杂赛道元素
- 角点识别，准确判断赛道特征

⚠️ **局限性**：
- 依赖角度阈值，需要针对场地调参
- 圆环处理复杂，状态转换条件苛刻
- 无法处理三岔路、急弯等特殊情况
- 对图像质量要求较高

---

### 8.2 改进方向

💡 **建议**：
1. **自适应阈值**：使用机器学习自动调整角点判断阈值
2. **预测模型**：引入卡尔曼滤波，预测赛道走向
3. **模糊控制**：替代硬性阈值判断，提高鲁棒性
4. **深度学习**：使用CNN直接回归转向角，简化流程

---
