## 项目概述

### 项目简介

本项目是一个基于LS2K0300龙芯处理器的**智能小车综合控制系统**，集成了以下核心功能：

- 🚗 **双轮差速驱动** - 基于DRV8701E电机驱动芯片的双电机控制
- 🎯 **速度闭环控制** - 增量式PID算法实现精确速度控制
- 📹 **视觉采集** - USB摄像头(UVC)实时图像采集
- 🖥️ **人机交互** - IPS200屏幕显示 + 4按键菜单系统
- 📡 **网络通信** - TCP图像传输与远程监控
- ⏱️ **实时调度** - 多线程实时任务调度系统
- 📊 **性能监控** - 纳秒级时间戳性能分析

### 应用场景

- 智能巡线小车
- 视觉导航机器人
- 远程监控移动平台
- 嵌入式控制系统教学

### 技术特点

1. **面向对象设计** - 采用C++17标准，模块化封装
2. **实时性保障** - 基于POSIX实时线程，优先级可配置
3. **跨平台兼容** - 基于Linux标准API，易于移植
4. **完善的调试接口** - 时间戳监控、性能分析、参数可调

## 模块详细设计

### 1. 电机控制模块 (motor.cpp/hpp)

#### 功能概述
封装双电机PWM+方向控制，提供速度设置和编码器速度采集功能。

#### 硬件映射
```cpp
// 编码器（方向编码器模式）
encoder1 → ZF_ENCODER_DIR_1  // 右轮编码器
encoder2 → ZF_ENCODER_DIR_2  // 左轮编码器

// 电机PWM和方向控制
right_motor → ZF_PWM_MOTOR_1     // 右轮PWM
left_motor  → ZF_PWM_MOTOR_2     // 左轮PWM
right_dir   → ZF_GPIO_MOTOR_1    // 右轮方向
left_dir    → ZF_GPIO_MOTOR_2    // 左轮方向
```

#### 核心参数
```cpp
// PWM限幅参数
#define MOTOR_PWM_MAX        7200    // 最大PWM值（10000为满占空比）
#define MOTOR_PWM_MIN        (-7200) // 最小PWM值
#define PWM_DEAD_ZONE        50      // PWM死区，消除零位抖动

// 速度映射参数（经实测标定）
#define ENCODER_MAX_PER_3MS  170     // 3ms最大编码器计数
#define TARGET_SPEED_MAX     120     // 控制器最大目标速度
#define PWM_PER_SPEED_UNIT   60      // 速度到PWM转换系数
#define REMAP_WEIGHT         1.0f    // 编码器计数权重
```

#### 核心函数

**1. motor_init()**
```cpp
功能：初始化电机PWM和方向GPIO
返回：bool - 初始化成功/失败
```

**2. motor_set_speed(left_speed, right_speed)**
```cpp
功能：设置双电机速度（PID输出 → PWM映射）
参数：
  - left_speed: -120~120（负值反转）
  - right_speed: -120~120
处理流程：
  1. 速度映射：速度 × PWM_PER_SPEED_UNIT → PWM值
  2. 死区处理：abs(PWM) < 50 → PWM = 0
  3. 硬限幅：PWM限制在±7200范围
  4. 方向+PWM输出：根据正负设置DIR和PWM
```

**3. get_and_remap_speed(&right_speed, &left_speed, sampling_period)**
```cpp
功能：读取编码器计数并映射为速度值
参数：
  - right_speed/left_speed: 输出速度变量指针
  - sampling_period: 采样周期(ms)，建议≤1000避免int16溢出
返回：bool - 采集成功/失败
处理流程：
  1. 读取编码器计数
  2. 速度映射（计数 × REMAP_WEIGHT）
  3. 清空编码器计数器
注意：必须在中断线程中定时调用
```

#### 技术要点
1. **线序修正** - 代码中left/right互换，因硬件接线反接
2. **死区设置** - 避免低速时电机无法启动或抖动
3. **速度映射** - 编码器计数值映射到PID可用的速度域
4. **采样周期** - 3ms采样周期经测试验证，既保证响应又避免计数溢出

### 2. PID控制器模块 (my_pid.cpp/hpp)

#### 功能概述
增强型增量式PID控制器，支持积分分离、微分滤波、反向控制等高级功能。

#### 核心特性
- ✅ 增量式PID算法
- ✅ 积分分离（抗积分饱和）
- ✅ 微分低通滤波
- ✅ 输出限幅
- ✅ 死区处理
- ✅ 先行项（Feedforward）
- ✅ 反向控制开关
- ✅ PID使能开关

#### 类设计
```cpp
class MyPID {
private:
    // 基础PID参数
    float Kp, Ki, Kd;             // PID三参数
    float Ti;                     // 积分时间常数
    float integral;               // 积分累积值
    float prev_error, last_error; // 误差历史
    float error_filter;           // 死区阈值
    
    // 输出限幅
    float output_max, output_min;
    float integral_max, integral_min;
    
    // 高级功能
    float derivative_filter;      // 微分滤波系数(0~1)
    float feedforward;            // 先行项
    bool  anti_windup;            // 积分分离开关
    float windup_zone;            // 积分分离死区
    
public:
    void init(...);               // 初始化PID参数
    float control(target, current); // PID计算
    void set_feedforward(float ff); // 设置先行项
    void enable_anti_windup(...); // 启用积分分离
    void set_derivative_filter(float alpha); // 设置微分滤波
    void reset();                 // 重置PID状态
};
```

#### 算法实现

**增量式PID公式**
```
output(k) = Kp * e(k) + Ki * sum(e) + Kd * [e(k) - e(k-1)]
```

**积分分离策略**
```cpp
if (abs(error) > windup_zone) {
    integral = 0;  // 超出死区，清空积分
} else {
    integral += error * Ti;  // 正常积分
}
```

**微分低通滤波**
```cpp
derivative_raw = Kd * (error - prev_error);
derivative_filtered = alpha * derivative_raw + 
                     (1-alpha) * derivative_last;
```

#### 参数调优指南

**初始化参数（实测标定值）**
```cpp
pid_r.init(
    1.5,    // Kp - 比例系数（响应速度）
    0.03,   // Ti - 积分时间常数（消除静差）
    0.1,    // Kd - 微分系数（抑制超调）
    1,      // error_filter - 死区阈值
    120,    // output_max - 输出上限
    -120,   // output_min - 输出下限
    60,     // integral_max - 积分限幅
    -60     // integral_min - 积分下限
);
```

**调参顺序建议**
1. **Kp先行** - 从小到大调，观察响应速度
2. **加入Ki** - 消除稳态误差，注意积分饱和
3. **加入Kd** - 抑制超调，提升稳定性
4. **限幅保护** - 防止输出过大损坏硬件


**2. encoder_get_count_handler()**
```cpp
功能：编码器采集线程回调
```

### 8. 主程序模块 (main.cpp)

#### 初始化流程
```cpp
int main() {
    // 1. 显示屏初始化
    ips200.init("/dev/fb0");
    ips200.clear();
    
    // 2. 编码器清零
    encoder1.clear_count();
    encoder2.clear_count();
    
    // 3. 电机初始化
    motor_init();
    motor_set_speed(0, 0);
    
    // 4. 摄像头初始化
    uvc.init("/dev/video0");
    
    // 5. PID初始化
    pid_r.init(1.5, 0.03, 0.1, 1, 120, -120, 60, -60);
    pid_l.init(1.5, 0.03, 0.1, 1, 120, -120, 60, -60);
    
    // 6. 菜单系统初始化
    menu_system.init_menu();
    
    // 7. 启动实时线程
    zf_driver_pit_rt key_scan;
    zf_driver_pit_rt encoder_get;
    zf_driver_pit_rt pid_control_thread;
    
    key_scan.init_ms(10, key_scan_handler, 98, true);
    encoder_get.init_ms(3, encoder_get_count_handler, 99, true);
    pid_control_thread.init_ms(30, pid_control_handle, 97, true);
    
    // 8. 主循环
    while(true) {
        menu_system.menu_system();
        system_delay_ms(20);
    }
}
```

#### 线程优先级分配
```
99 (最高) → 编码器采集 (3ms周期，最关键)
98        → 按键扫描   (10ms周期，用户交互)
97        → PID控制    (30ms周期，控制输出)
默认      → 主线程     (菜单显示)
```

## 技术要点

### 1. 实时性保障

#### 线程优先级设计原则
```
编码器采集(99) > 按键扫描(98) > PID控制(97)
```
- 编码器采集周期最短(3ms)，优先级最高
- PID控制周期为编码器10倍，优先级次之
- 按键扫描可容忍延迟，优先级较低

#### POSIX实时调度
```cpp
struct sched_param param;
param.sched_priority = priority;
pthread_setschedparam(pthread_self(), SCHED_FIFO, &param);
```

### 2. 速度映射标定

#### 映射关系
```
编码器计数 → 实际速度 → PID目标速度 → PWM输出
```

#### 实测数据（3ms采样周期）
```
PWM=7200 → 编码器计数≈170 → 定义为速度120
PWM=3600 → 编码器计数≈85  → 定义为速度60
```

#### 映射公式
```cpp
// 编码器→速度
speed = encoder_count * REMAP_WEIGHT;

// 速度→PWM
pwm = speed * PWM_PER_SPEED_UNIT;
```

### 3. PID调参经验

#### 参数对系统影响
```
Kp ↑ → 响应快，超调大
Ki ↑ → 消除静差，易饱和
Kd ↑ → 抑制超调，抗干扰
```

#### 本项目参数
```cpp
Kp = 1.5   // 中等响应速度
Ki = 0.03  // 小积分，避免饱和
Kd = 0.1   // 小微分，抑制抖动
```

#### 调参注意事项
1. 积分限幅防饱和：integral_max = 60
2. 输出限幅保护：output_max = 120
3. 死区阈值：error_filter = 1
4. 微分滤波：derivative_filter = 0.3

### 4. 线程安全

#### 全局变量访问
```cpp
// 只读变量（编码器线程写，其他线程读）
float right_speed;  // 无需互斥锁
float left_speed;

// 可写变量（多线程访问）
float target_speed_r;  // 需注意原子性
```

#### 建议
- 简单数值型变量（float/int16）可直接访问
- 复杂结构体或数组考虑使用互斥锁
- 中断线程尽量避免阻塞操作

### 5. 硬件接线注意事项

#### 电机接线反接问题
```cpp
// 代码中left/right互换
if (right_pwm <= 0) {

}
```
